# 表单设计器模块设计说明

本文档用于说明 `designer` 目录下各模块的职责边界与设计思路，帮助后续迭代时快速定位代码。

## 一、总体设计思路

- **入口职责收敛**：`index.vue` 仅作为路由入口壳层，实际页面实现收口在 `DesignerPage.vue`。
- **按领域拆分逻辑**：围绕“画布、属性、差异、持久化、撤销重做、选择”等垂直领域拆成 composable。
- **低耦合高内聚**：每个 composable 只处理一类问题，通过 `Ref` 与回调函数协作，避免互相直接依赖。
- **行为保持优先**：拆分过程优先保持原交互与校验语义不变，再提升结构可读性。

---

## 二、入口与页面层

### `index.vue`

- **职责**：路由入口与兼容壳层，仅挂载 `DesignerPage.vue`。
- **设计重点**：保持入口稳定，避免路由层与业务实现耦合。

### `DesignerPage.vue`

- **职责**：当前表单设计器页面主实现（模板编排与交互接入）。
- **设计重点**：对外保持稳定页面能力，对内通过状态层/composable 持续拆分。

---

## 三、状态与持久化

### `useDesignerPageState.ts`

- **职责**：当前页面状态聚合层，集中管理字段编排、规则编辑、画布交互与差异相关状态。
- **设计思路**：
  - 作为历史实现的统一状态出口，保障 `DesignerPage.vue` 的行为稳定。
  - 逐步将可独立能力下沉到细粒度 composable，降低单文件复杂度。

### `useDesignerSnapshots.ts`

- **职责**：快照、撤销/重做、恢复状态、去重与容量控制。
- **设计思路**：
  - 通过 `signature` 判断状态是否变化，避免无效入栈。
  - 恢复状态时统一设置 `restoring`，防止 watch 触发连锁回写。

### `useDesignerPersistence.ts`

- **职责**：加载详情、保存、导入/导出、返回列表。
- **设计思路**：
  - 把 API 交互与页面交互（消息提示）放到同一层，减少入口文件负担。
  - 对旧数据兼容（如 published/draft 兼容读取）集中在这里。

---

## 四、画布与分组

### `useDesignerCanvasGrouping.ts`

- **职责**：
  - `schema` 与 `canvasBuckets` 双向结构转换。
  - 分组维护（新建/重命名/删除/折叠）。
  - 画布搜索可见性统计（组内可见数量、整体可见数量）。
- **设计思路**：
  - 将“结构组织”与“业务字段属性”解耦。
  - 分组操作方法返回 `boolean` 表示是否有实际变更，方便上层决定是否写快照。

### `useDesignerSync.ts`

- **职责**：拖拽后同步逻辑与 watch 同步（`schema <-> buckets`）。
- **设计思路**：
  - 专门处理“同步副作用”，避免入口文件充满 watch 噪音。
  - 通过 `restoring` 防止恢复快照时触发重复同步。

### `useDesignerPalette.ts`

- **职责**：组件库过滤、克隆字段模板、唯一字段名生成。
- **设计思路**：
  - “拖入画布时创建字段”是独立能力，不应散落在入口文件。
  - 统一默认字段模板，保证新增字段行为一致。

---

## 五、字段编辑与规则

### `useDesignerCurrentField.ts`

- **职责**：当前字段定位与字段 patch 写回（包含分组变更桥接）。
- **设计思路**：
  - 把“当前字段上下文”单独封装，降低属性面板相关代码耦合。

### `useDesignerFieldProps.ts`

- **职责**：
  - 属性面板核心逻辑（支持能力判断、选项维护、正则预设）。
  - `dataSource/cascader/componentProps` 三类 JSON 文本编辑与格式化。
  - 联动条件相关辅助（操作符、选项、条件值归一）。
- **设计思路**：
  - 按“属性编辑域”集中所有细节，避免入口文件变成巨型脚本。
  - 文本编辑采用“容错输入 + 延迟快照”的交互策略，减少打断。

### `useDesignerCrossRules.ts`

- **职责**：跨字段规则 JSON 文本模式的解析与校验。
- **设计思路**：
  - 独立维护 JSON 校验语义，保证错误提示文案和逻辑稳定。

### `useDesignerCrossFieldBindings.ts`

- **职责**：跨字段面板桥接（字段选项、advanced 开关、规则文本更新）。
- **设计思路**：
  - 把“面板事件转状态更新”的胶水层统一放一起，减少入口零散函数。

### `useDesignerQuality.ts`

- **职责**：质量检查聚合（空字段、重名、循环依赖、跨字段规则完整性等）。
- **设计思路**：
  - 质量检查视为独立“诊断域”，不与保存逻辑混写。
  - 输出标准问题结构，供 UI 统一渲染与定位。

---

## 六、交互与操作

### `useDesignerSelection.ts`

- **职责**：单选/多选/区间选、批量操作、按 uid 删除。
- **设计思路**：
  - 把“选择状态机”从页面中剥离，便于后续扩展（例如框选、全选）。

### `useDesignerCanvasOps.ts`

- **职责**：预览校验、字段移动/复制/删除、快捷键（撤销重做）。
- **设计思路**：
  - 聚焦“动作级”逻辑，不关心画布组织细节。
  - 与 `useDesignerSelection`、`useDesignerSnapshots` 通过最小接口协作。

### `useDesignerDiff.ts`

- **职责**：草稿与已发布的字段/分组差异计算与抽屉状态管理。
- **设计思路**：
  - 差异算法独立，便于后续扩展更多 diff 维度。

---

## 七、子组件分工

### `DesignerBasicPropsPanel.vue`
- 负责基础属性项（标题、字段名、占位、选项等）展示与事件抛出。

### `DesignerAdvancedPropsPanel.vue`
- 负责高级配置区域（联动、透传 props）展示与事件抛出。

### `DesignerCrossRulesPanel.vue`
- 负责跨字段规则可视化/JSON 模式 UI 展示与事件抛出。

### `DesignerCanvasItem.vue`
- 负责画布字段卡片渲染与交互事件（选中、删除）。

---

## 八、后续维护建议

- 新需求优先判断归属到哪个 composable，再实现，避免逻辑继续堆积到 `DesignerPage.vue` / `useDesignerPageState.ts`。
- 新增面板事件时，优先在“bindings 类 composable”补桥接函数。
- 涉及 `schema` 与 `canvasBuckets` 同步的改动，优先在 `useDesignerSync` 与 `useDesignerCanvasGrouping` 内处理。
- 涉及规则/校验文案的改动，集中在 `useDesignerCrossRules` 与 `useDesignerQuality`，保证行为一致。
