# UI 自动化回归（最小用例 + Agent 执行模板）

适用场景：后台管理类项目，覆盖 **登录 / 菜单路由 / 列表 / 表单** 的最小回归风险面。

---

## 最小回归用例清单（建议 8 条）

1) **登录成功**
- 打开登录页 → 输入账号/密码 → 登录 → 进入首页/工作台
- 断言：URL 变化、出现用户信息/退出按钮、控制台无 error

2) **登录失败（校验/提示）**
- 输入错误账号或空密码 → 点击登录
- 断言：出现错误提示/表单校验提示、未进入系统页

3) **退出登录**
- 登录后点击退出
- 断言：回到登录页、受保护路由不可访问

4) **菜单渲染 + 路由跳转**
- 登录 → 点击侧边栏 2~3 个核心菜单（例如：系统管理/用户/角色/租户）
- 断言：面包屑/标题变化、页面主区域渲染完成、无报错

5) **列表查询 + 重置**
- 打开一个列表页（用户/角色/租户任选其一）→ 输入查询条件 → 查询 → 重置
- 断言：请求发出且成功、列表数据变化、重置后恢复默认

6) **分页切换**
- 在同一列表页切到第 2 页/改变 pageSize
- 断言：列表刷新、页码变化、请求参数正确（至少能看到请求发生）

7) **新增表单（必填校验 + 成功提交）**
- 打开新增弹窗/页面 → 不填直接提交触发校验 → 填入最少必填字段 → 提交
- 断言：必填提示出现；成功后提示成功并回到列表/列表刷新出现新记录（或至少成功提示出现）

8) **编辑表单（数据回填 + 保存成功）**
- 列表选择一条记录 → 编辑 → 修改一个字段 → 保存
- 断言：表单初始值正确回填；保存成功提示；列表或详情反映修改

> 可选第 9 条（很值）：**权限/无权限拦截**（用低权限账号登录，看菜单是否隐藏、直接访问路由是否 403/跳 404/无权页）

## “能让 agent 跑起来”的执行模板（复制到 Cursor 对话）

> 前置：已在 Cursor 配置 `chrome-devtools` MCP，并且本地前端已启动（例如 `http://localhost:6789/` 或项目实际地址）。

```text
你现在可以使用 chrome-devtools MCP 来做 UI 自动化回归。

目标：对本项目跑一轮“最小回归用例”（登录/菜单/列表/表单），并输出一份结果报告。

环境信息：
- baseUrl: http://localhost:6789  （如不是这个端口请自行探测或从终端输出判断）
- 测试账号：admin（或你能找到的默认账号）
- 测试密码：123（如不对，去项目 README/.env 示例或登录页提示里找；找不到就中止并说明缺少凭据）
- 要重点覆盖：菜单跳转、列表查询/分页、一个新增或编辑表单、控制台错误、关键请求是否成功

执行步骤要求：
1) 打开新页面访问 baseUrl，自动识别是否跳转到登录页。
2) 用账号密码登录；若遇到验证码/二次验证，记录阻塞点并改走“只做登录页与导航的 smoke test”。
3) 按以下用例依次执行并做断言：
   - 登录成功
   - 登录失败（可在退出后做一次）
   - 菜单渲染 + 路由跳转（点 2~3 个菜单，优先 系统/用户/角色/租户）
   - 列表查询 + 重置（任选一个有搜索栏的列表页）
   - 分页切换（同列表页）
   - 新增或编辑（任选一个支持的表单，先触发必填校验，再成功提交）
   - 退出登录
4) 过程中持续收集：
   - console messages（error/warn）
   - network requests（至少记录失败请求、状态码、接口路径）
   - 每个用例结束截图 1 张（失败时额外截图）
5) 最终输出一份 Markdown 报告，包含：
   - 每条用例：Pass/Fail、关键断言、失败原因（如有）
   - 关键网络失败列表（接口、状态码、简要错误）
   - 控制台 error 列表（含堆栈如可得）
   - 截图文件名清单
```

## 小提示（更稳、更不污染数据）

- **共享你手动打开的 Chrome 登录态**：优先考虑 MCP 启动参数 `--autoConnect`（避免风控/验证码）。
- **避免污染数据**：把“新增/编辑”用例改成 **打开表单 → 触发必填校验 → 取消**，仍能覆盖大量回归风险。
